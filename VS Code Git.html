VS Code Git

/// ทำได้ทั้งผ่าน UI (Source Control panel) และ Terminal — เลือกตามสะดวก

1. เปิด Project และ git init

VS Code UI
File → Open Folder → เลือก folder project
Source Control Ctrl+Shift+G → คลิก Initialize Repository
Terminal 
git init

/// Terminal ใน VS Code: Ctrl+` (backtick)

2. git add และ git commit

VS Code UI
Source Control panel Ctrl+Shift+G
ไฟล์ที่แก้จะขึ้นใต้ Changes
กด + ข้างไฟล์ = git add (ย้ายไป Staged Changes)
พิมพ์ commit message ช่องบน → กด Ctrl+Enter = git commit
Terminal 
git add main.py          # add เฉพาะไฟล์
git add .               # add ทุกไฟล์
git commit -m "message"
UIเทียบกับ Terminal
Changes → กด +git add <file>
Stage All Changesgit add .
พิมพ์ message → Ctrl+Entergit commit -m "..."

3. .gitignore

VS Code UI
Explorer → คลิกขวา → New File → ตั้งชื่อ .gitignore
พิมพ์ชื่อไฟล์/folder ที่ต้องการ ignore 
secret_key.txt
.venv/
.idea/
GitCore/

/// VS Code จะแสดงไฟล์ที่ถูก ignore เป็นสีเทาใน Explorer อัตโนมัติ

4. ดู Git Log

Terminal 
git log --oneline
Extension: Git Graph
Extensions Ctrl+Shift+X → ค้นหา Git Graph → Install
Source Control panel → คลิก Git Graph icon มุมบนขวา
จะเห็น branch tree แบบ visual พร้อม commit history

/// Git Graph แสดงผล branch และ merge ได้ชัดกว่า log --oneline มาก

5. git diff — ดูการเปลี่ยนแปลง

VS Code UI
Source Control → คลิกที่ชื่อไฟล์ใน Changes
VS Code จะเปิด diff editor เปรียบเทียบซ้าย/ขวาอัตโนมัติ
สีเขียว = เพิ่ม    สีแดง = ลบ
Terminal 
git diff                    # unstaged
git diff --staged           # staged
git diff abc1234 def5678    # เปรียบเทียบ 2 commits

6. สร้างและจัดการ Branch

VS Code UI
มุมล่างซ้าย คลิกชื่อ branch (เช่น main)
→ Create new branch → พิมพ์ชื่อ → Enter
→ VS Code จะ switch ไปที่ branch ใหม่อัตโนมัติ
Terminal 
git branch feature/rename-game      # สร้าง
git switch feature/rename-game      # ย้ายไป
git switch -c feature/rename-game   # สร้าง + ย้าย
git branch                          # ดู branch ทั้งหมด
git branch -d feature/rename-game   # ลบ (merge แล้ว)
git branch -D feature/rename-game   # ลบ force

/// ชื่อ branch ปัจจุบันแสดงมุมล่างซ้ายของ VS Code เสมอ

7. Merge Branch

VS Code UI
Ctrl+Shift+P → พิมพ์ Git: Merge Branch
→ เลือก branch ที่ต้องการ merge เข้า branch ปัจจุบัน
Terminal 
git switch main
git merge feature/rename-game
Fast-forward = merge สะอาด ไม่มี conflict
CONFLICT = ต้องแก้เอง

8. แก้ Merge Conflict

VS Code UI
ไฟล์ที่มี conflict จะขึ้นสัญลักษณ์ ! ใน Source Control
คลิกเปิดไฟล์ → VS Code แสดงปุ่มให้เลือก:

Accept Current Change → เอาของ HEAD (branch ปัจจุบัน)
Accept Incoming Change → เอาของ branch ที่ merge เข้ามา
Accept Both Changes → เอาทั้งคู่
Compare Changes → เปิด diff editor
Terminal 
git diff                          # ดู conflict marker
# แก้ไฟล์ — ลบ <<<<<<< === >>>>>>> ออก
git add Hex_Heroes.html
git commit -m "Resolve conflict"
<<<<<<< HEAD
<title>Hex Arena Online</title>
=======
<title>Wuxia Hex Arena v2.0</title>
>>>>>>> feature/rename-game-b2

/// ใน VS Code ไม่ต้องแก้ marker เอง — คลิกปุ่มด้านบนไฟล์ได้เลย

9. Push ขึ้น GitHub

VS Code UI
Source Control → คลิก ⋯ (More Actions) → Push
หรือคลิกปุ่ม Sync Changes (ลูกศรวนกลม) มุมล่างซ้าย
Terminal 
git push                          # push (หลังตั้ง upstream แล้ว)
git push -u origin main           # push ครั้งแรก + ตั้ง upstream

/// VS Code จะแจ้งเตือนถ้ายังไม่ได้ login GitHub — คลิก Sign in ได้เลย
